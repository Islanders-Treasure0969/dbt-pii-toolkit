# Security Policy

## サポート対象バージョン

| バージョン | 脆弱性パッチ |
|------------|--------------|
| 0.x (latest) | ✅ サポート中 |
| 0.x (older minor) | ❌ サポート対象外。最新 minor へのアップグレードを推奨 |

v1.0 リリース後は LTS ポリシーを別途定義します。

## 脆弱性の報告

**公開イシューには絶対に投稿しないでください。** 攻撃者に悪用される前に修正したいので、
以下のいずれかで非公開に報告してください:

1. **GitHub Security Advisory** (推奨): このリポジトリの "Security" タブから "Report a vulnerability"
2. **Email**: hyuujm821@gmail.com (件名に `[dbt-pii-toolkit:security]` を含めてください)

報告に含めてほしい情報:

- 影響を受けるバージョン
- 再現手順 (合成データのみ使用)
- 想定される影響範囲
- 既知の回避策があれば

72時間以内に一次返信します。修正リリースまでの目安は **重大度 High 以上で14日以内**です。

## 対応プロセス

1. 受領確認 (24-72時間)
2. 影響範囲評価 + CVSS スコアリング
3. 修正パッチ作成 + private 検証
4. リリース + GitHub Security Advisory 公開
5. 報告者へクレジット (希望する場合)

## サンプルデータ方針 ⚠️

このリポジトリ内のすべてのデータは **合成データ (synthetic data)** です。
実在する個人・法人のデータを含めることは **厳禁** です。

PR を提出する際は、以下を確認してください:

- ❌ 本番データのコピーをサンプルにしない
- ❌ メンバーシップ攻撃で復元可能な準合成データを使わない
- ✅ Faker / Mockaroo 等で生成した完全合成データのみ使用
- ✅ 再現性のため seed を固定 (`seed=42` 等)

合成データであっても、レアな組み合わせ (例: 特定の生年月日 + 郵便番号 + 性別) は
個人を特定可能になることがあります。`k-anonymity ≥ 5` 程度を目安にしてください。

## 既知の制約事項

- **決定的ハッシュ** (`hash_deterministic`) はソルトなしで使うとレインボーテーブル攻撃に脆弱です。
  必ずプロジェクト固有のソルトを `var('pii_salt')` で渡してください。
- **部分マスキング** (`mask_email`, `mask_phone`) は **可逆ではない** が、
  低エントロピーの値については周辺情報との突合で再特定される可能性があります。
- このマクロ集は **適合性判断の代替にはなりません**。法務部門の確認を必ず通してください。

## 暗号化キー / シークレット管理

このリポジトリは秘密情報を扱う場合、以下の方針に従ってください:

- ✅ `var('pii_salt')` 等で外部から注入
- ✅ Snowflake → AWS KMS / Snowflake内部キー管理
- ✅ BigQuery → Cloud KMS + CMEK
- ❌ ハードコードされたソルト / キーを commit しない (CI で `gitleaks` 等の検査を予定)
